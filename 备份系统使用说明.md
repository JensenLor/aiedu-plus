# aiedu.plus 页面文件备份系统使用说明

## 概述

为了防止我在修改网站页面时出现错误，主公建议建立版本备份系统。本系统专门为aiedu.plus网站的HTML/CSS页面文件设计，提供完整的版本控制和恢复功能。

## 备份文件范围

备份系统会自动备份以下页面文件：
- `index.html` - 首页
- `works.html` - MakerVerse作品展示页
- `work-detail.html` - 作品详情页
- `resources.html` - 学习资源页
- `cases.html` - 学生列表页
- `profile.html` - 学生个人主页
- `preview.html` - 预览页
- `test-work-detail.html` - 测试页
- `style.css` - 样式文件

## 备份工具说明

### 1. `backup_pages.bat` - 创建备份
**功能**：创建当前页面文件的备份版本
**使用**：双击运行，自动创建带时间戳的备份
**输出**：在`backups/`目录下创建`version_YYYYMMDD_HHMMSS`目录

### 2. `backup_list.bat` - 查看备份列表
**功能**：显示所有可用备份版本
**使用**：双击运行，查看备份历史
**输出**：列出所有备份版本、时间和文件数量

### 3. `restore_version.bat` - 恢复备份
**功能**：恢复指定版本的页面文件
**使用**：双击运行，选择版本编号恢复
**特性**：恢复前会自动创建当前状态的备份，确保安全

### 4. `backup_cleanup.bat` - 清理旧备份
**功能**：自动清理旧备份，保留最近10个版本
**使用**：双击运行，确认清理操作
**策略**：只保留最新10个版本，避免磁盘空间占用

## 工作流程

### 正常开发流程
1. 开发前运行 `backup_pages.bat` 创建备份
2. 进行页面修改和开发
3. 测试修改效果
4. 如果需要回滚，运行 `restore_version.bat`

### 错误恢复流程
1. 发现页面错误或误改
2. 运行 `backup_pages.bat` 备份当前错误状态（可选）
3. 运行 `backup_list.bat` 查看可用版本
4. 运行 `restore_version.bat` 恢复正确版本
5. 验证恢复结果

### 定期维护流程
1. 定期运行 `backup_cleanup.bat` 清理旧备份
2. 重要版本可手动复制到其他位置永久保存

## 备份策略

### 时间戳命名
每个备份版本使用时间戳命名：`version_YYYYMMDD_HHMMSS`
- YYYY: 年
- MM: 月
- DD: 日
- HH: 时
- MM: 分
- SS: 秒

### 保留策略
- 自动保留最近10个版本
- 旧版本自动清理
- 恢复前的状态会自动备份（安全机制）

### 备份目录结构

## 使用建议

### 何时创建备份
1. **重大修改前** - 修改核心功能或布局时
2. **批量修改前** - 修改多个页面时
3. **实验性功能开发前** - 尝试新功能时
4. **定期备份** - 每周或每月创建里程碑版本

### 何时恢复备份
1. **出现明显错误时** - 页面显示异常或功能失效
2. **误删重要内容时** - 不小心删除了重要代码
3. **测试失败时** - 新功能测试不通过
4. **用户反馈问题时** - 用户报告页面问题

### 安全注意事项
1. 恢复前会自动备份当前状态，提供双重保障
2. 重要版本建议手动复制到项目外部
3. 定期检查备份目录，确保磁盘空间充足

## 自动化建议

主公可以在我每次修改页面文件前要求我运行备份，或者在发现错误时立即恢复。我也会在记忆系统中记录备份事件：

```memory
重要操作前备份：修改 [页面名称] 前创建版本 [version_XXXX]
错误恢复：从版本 [version_XXXX] 恢复 [页面名称]
```
