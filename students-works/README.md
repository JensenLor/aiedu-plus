# 学生作品数据管理系统

## 系统概述

这是一个面向非技术人员的学生作品管理系统，允许学生和教师通过简单的CSV表格和文件夹操作来管理作品展示。系统已优化，移除手动维护字段，实现自动计算。

## 核心特点

1. **零编程门槛**：使用CSV表格管理数据，Excel可直接打开编辑
2. **自动计算**：学习时长自动从加入日期计算，作品数量自动统计
3. **直接下载**：作品可直接下载ZIP包，无需跳转GitHub
4. **每周更新**：数据每周自动同步一次
5. **动态展示**：网站实时加载最新JSON数据

## 快速开始

### 第一步：添加新学生

1. 用Excel打开`学生基本信息.csv`
2. 在最后一行添加学生信息（学号、姓名、年级等）
   - **学习时长和作品数量无需填写**：系统自动计算
   - **加入日期**：填写实际加入日期（格式：YYYY-MM-DD）
3. 在`students/`目录创建文件夹：`学号_姓名`（如：`zzy001_曾梓扬`）

### 第二步：上传作品

1. 在学生文件夹的`works/`子目录下创建文件夹：`作品ID_作品名称`
2. 将作品文件放入该文件夹：
   - `README.txt` - 作品详细说明
   - `preview.txt` - 预览摘要（100字内）
   - 项目源代码文件（可选）
   - 图片/视频文件（可选）
3. 用Excel打开`作品信息.csv`，添加作品信息
   - **GitHub链接无需填写**：系统自动生成ZIP下载包
   - **下载文件字段自动填充**：运行ZIP打包脚本后自动更新

### 第三步：生成数据包

运行脚本生成网站数据：
```
# 方法1：使用批处理一键生成
create_zip.bat

# 方法2：分步执行
# 第一步：生成ZIP下载包
python scripts/create_zip_packages.py

# 第二步：更新JSON数据
python scripts/simple_update_data.py
```

### 第四步：查看网站

直接访问网站，系统会自动加载最新JSON数据，无需手动更新HTML页面。

## 故障排除

### 常见问题

1. **CSV文件无法读取**
   - 检查文件是否为`.csv`格式（Excel可直接打开）
   - 确认编码为UTF-8（Excel保存时选择"CSV UTF-8"）
   - 确保逗号分隔符正确

2. **学生卡片未显示**
   - 检查学生文件夹命名：`学号_姓名`
   - 确认CSV中有对应的学生信息
   - 查看`generated/students.json`是否生成
   - 检查浏览器控制台是否有加载错误

3. **ZIP包无法下载**
   - 确认已运行`create_zip.bat`或`create_zip_packages.py`
   - 检查`downloads/`目录是否有对应的`.zip`文件
   - 确认作品文件夹中有文件（空文件夹无法生成ZIP）

4. **学习时长计算错误**
   - 检查"加入日期"格式：YYYY-MM-DD
   - 确认日期有效（不超过当前日期）
   - 系统每月自动重新计算

5. **脚本运行失败**
   - `simple_update_data.py`：纯Python，无需安装依赖
   - `create_zip_packages.py`：需要Python标准库
   - `create_zip.bat`：需要Windows PowerShell 5.0+

## 🎯 学号的重要作用

### 为什么必须保留学号？
1. **唯一标识**：每个学生在系统中的唯一ID（防止同名问题）
2. **数据关联**：学生信息 ↔ 作品信息 ↔ 文件夹的关联键
3. **查找依据**：通过学号快速定位学生所有作品
4. **系统一致性**：文件夹命名、数据查询、页面展示的统一基础

### 学号在系统中的角色
- **文件夹命名**：`zzy001_曾梓扬/`
- **CSV关联**：学生基本信息.csv ↔ 作品信息.csv
- **JSON生成**：学生数据与作品数据的关联依据
- **网站查询**：点击学生卡片时传递学号参数

### 如果学号被移除的影响
1. **数据无法关联**：不知道作品属于哪个学生
2. **文件夹混乱**：无法按规范命名文件夹
3. **查询困难**：无法快速查找学生的所有作品
4. **系统崩溃**：整个数据关联体系失效

**结论：学号是系统的骨架，必须保留且唯一。**
